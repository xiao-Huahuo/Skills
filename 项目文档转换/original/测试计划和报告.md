## 1. 引言

### 1.1 编写目的

本文档是为 **“MetaWeave（元织）”** 软件系统编写的测试计划和报告书。本文件的核心目的在于：

1. **规范测试活动**：明确测试范围、测试内容、测试策略、资源安排和进度计划，指导测试团队有序开展工作。
2. **验证需求达成**：确保软件产品满足《需求规格说明书》中定义的功能需求、非功能需求和外部接口需求。
3. **缺陷管理与质量评估**：系统记录测试过程中发现的问题，评估产品质量，为产品发布提供决策依据。
4. **过程改进**：通过对测试结果的分析，发现开发过程中的薄弱环节，为后续迭代提供改进建议。

**预期读者**：测试人员、开发人员、项目经理、产品质量负责人、运维团队、最终用户代表。

### 1.2 背景

- **被测项目名称**：MetaWeave（元织）个人知识中枢
- **开发目的**：解决数字化办公时代知识资产碎片化、检索困难、信息孤岛等核心痛点，实现"缝合碎片，编织智慧"的愿景。
- **功能需求概述**：
  - 智能化增强功能：语义意图搜索、交互式关系图谱、OCR图片文字识别、任务资料聚合、隐私脱敏、智能分类建议、基础及高级智能体等。
  - 基本操作与自动化功能：本地文件实时同步、多设备同步、网页抓取、动态热度管理、基础文件操作、多用户协作、存证导出等。
- **非功能需求概述**：
  - 性能：语义检索首字返回时间 ≤ 200ms，万级文件去重扫描 < 2秒，图谱渲染30fps以上。
  - 安全性：TLS 1.2+传输加密，bcrypt密码哈希，AES-256文件加密，多租户数据隔离。
  - 鲁棒性：7x24小时稳定运行，多级解析Pipeline容错。
  - 易用性：核心操作3步内完成。
- **环境约束**：
  - 前端：Vue 3 + Vite + Electron
  - 后端：Python 3.10+ + FastAPI + Celery
  - 数据库：PostgreSQL/SQLite + ChromaDB + Redis
  - 运行环境：现代浏览器或Electron桌面应用，服务器建议8核16GB以上。

### 1.3 定义

| 术语/缩写       | 定义                                                     |
| :-------------- | :------------------------------------------------------- |
| **RAG**         | 检索增强生成，通过检索私有知识库增强大模型输出准确性     |
| **Agent**       | 智能体，具备感知、决策和工具调用能力的AI逻辑单元         |
| **OCR**         | 光学字符识别，将图片中的文字转化为可编辑文本             |

| **CoT**         | 思维链，展示AI推理过程的技术                             |
| **DFA**         | 确定有穷自动机，用于高效敏感词过滤                       |

## 2. 测试计划

### 2.1 软件说明

#### 2.1.1 软件需求概述

MetaWeave是一款从底层文件管理进化而来的个人知识中枢，核心需求包括：

- **全格式内容感知**：通过OCR和流式解析，使PDF、图片、代码等文件不再是"内容黑盒"。
- **智能知识大脑**：利用RAG和Agent技术，自动编织语义关联的知识图谱。
- **主动式服务**：实现风险预警、重要日期提醒等智能自驱动功能。
- **安全可控的数据平台**：内置隐私脱敏、多端同步、存证导出等功能。

#### 2.1.2 软件体系结构设计简述

- **前端**：Vue 3 + Vite构建Web端，Electron打包跨平台桌面应用。
- **后端**：FastAPI异步应用服务器，提供RESTful API和WebSocket。
- **异步层**：Redis + Celery处理OCR、LLM推理、Embedding等耗时任务。
- **数据层**：SQLite/PostgreSQL（关系型）+ ChromaDB（向量型）双引擎。
- **AI层**：LangChain/LangGraph框架，支持多模型路由（Kimi、GPT-4o、DeepSeek）。

#### 2.1.3 测试环境要求

| 环境类型           | 配置要求                                                     |
| :----------------- | :----------------------------------------------------------- |
| **功能测试环境**   | CPU 4核，RAM 8GB，Ubuntu 22.04，Python 3.10，Node.js 18+     |
| **性能测试环境**   | CPU 8核，RAM 16GB，NVIDIA GPU（显存≥4GB），SSD存储           |
| **客户端测试环境** | Windows 11 / macOS 14 / Ubuntu 22.04，Chrome/Firefox/Safari最新版 |
| **网络环境**       | 有线网络（低延迟<10ms），模拟弱网（3G/4G，延迟100-300ms）    |
| **依赖服务**       | LLM API Mock服务或沙箱环境，MinIO对象存储，Redis 7.x，ChromaDB 0.4+ |

#### 2.1.4 测试范围

本次测试覆盖以下内容：

- **功能测试**：所有P0、P1优先级功能（共约20项），部分P2功能。
- **性能测试**：语义检索、去重扫描、图谱渲染、文件上传下载。
- **界面测试**：UI布局、交互组件、快捷键、响应式适配。
- **压力测试**：并发用户数、批量文件处理、WebSocket连接数。
- **健壮性测试**：异常输入、服务中断、网络波动、资源不足。
- **兼容性测试**：主流操作系统、浏览器、文件格式。
- **安全测试**：认证授权、数据加密、越权访问、XSS/CSRF防护。

**不包含范围**：P3优先级功能（如财务账单审计、文档趋停自动总结）、特定行业垂直解决方案。
