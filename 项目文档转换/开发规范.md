# 开发规范
### 前端
1. 所有组件均需划分为页面组件与子组件,分别放置在`src/views`与`src/components`目录下,页面组件需使用大驼峰命名法,子组件应当存储于相应的页面的蛇形命名法目录下,并使用大驼峰命名法命名组件文件,组件文件内的组件名称也应当与文件名保持一致,如:
```
src/
├── views/                  # 页面组件（路由直接指向的组件）
│   ├── UserProfile.vue     # 大驼峰命名
│   └── HomeView.vue
│
└── components/             # 子组件（被页面或其它组件引用的部分）
    ├── user_profile/       # 对应页面的蛇形命名文件夹
    │   ├── UserInfo.vue    # 大驼峰命名，与文件名一致
    │   └── SecuritySet.vue
    └── common/             # 全局通用组件
        └── CustomButton.vue
```
2. 所有API路由与请求拦截器均登记于`src/router/api_routes.js`文件中,页面路由和路由守卫和vue-router相关的逻辑均登记于`src/router/index.js`中,以便于统一管理和维护。禁止自己乱放路由.
3. 全局通用的颜色(用于全局颜色切换等)放置于`src/assets/main.css`中.
4. 登录注册页面统一使用`src/views/LoginOrRegister.vue`组件,禁止自己编写其他的登录或者注册页面.
5. 所有的组件应保持高可复用性,能复用绝不重写,能抽象绝不重复,能组件化绝不冗余.
6. 项目UI和配色应该模仿GitHub Desktop,保持简洁和专业,尽量使用白色或者黑色,以及绿色或者红色按钮,浅蓝色条幅或文件夹标志或者链接,避免使用其他颜色.
### 后端
1. 所有常量以及环境变量均由`app.core.config`模块的`GlobalConfig`类统一管理,禁止在代码中直接编写全局通用的常量,必须通过引入该模块来获取常量值,以便于统一管理和维护.
2. 模型层规范:分为`app.models`和`app.schemas`两个模块,前者存放XXXBase(基础模型)和XXX(数据库模型),后者存放三个DTO(如:XXXCreate(创建DTO,继承XXXBase),XXXUpdate(更新DTO,继承SQLModel),XXXOut(响应DTO,继承XXXBase)).必要时可以适当变通.
3. Log日志存放于根目录的`logs/app.log`文件中,可以调用`app.core.logger`模块的`global_logger`对象记录日志,如下:
```python
from app.core.logger import global_logger as logger
async def login():
    logger.info("用户尝试登录")
    try:
        # ... 业务逻辑
        logger.info("登录逻辑")
    except Exception as e:
        logger.error(f"登录失败: 请检查用户名和密码是否正确") 
```
4. 如果业务繁杂(如复杂的数据处理与算法),则应该从接口层中抽象为函数,并放置在业务层`app.services`或者AI业务层`app.ai`.
